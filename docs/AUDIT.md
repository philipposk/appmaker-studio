# AppMaker вЂ” Audit & Remediation Log

Post-migration audit (Supabase + Vercel, 6x7 platform) and the fixes applied on
branch `platform-fixes`. Severity reflects impact on the core product.

## Executive summary

- **The generate в†’ view в†’ refine loop was silently broken.** Generation saved
  files in a flat shape no UI component could read; after save the Code tab,
  Live Preview, Workflow graph and refine-context all came up empty. **Fixed.**
- **The database layer is healthy** (verified live): `appmaker.apps` /
  `appmaker.iterations` have RLS with correct owner-scoped policies, `anon` has
  no access, and the Supabase security advisor flags zero AppMaker issues. The
  schema is exposed to the API; authenticated users have full access. There was
  **no schema-exposure blocker** вЂ” an early assumption that proved false.
- **Build is green** throughout; every issue was a logic/architecture/perf
  defect, not a compile failure.

### Dismissed (verified false positives)
- `deleteCookie` omitting Secure/SameSite does **not** break deletion (RFC 6265:
  delete identity is name+Domain+Path).
- OAuth callback `<Navigate>` does **not** drop the session вЂ” exchange runs at
  module load; History API navigation can't interrupt it.

## Build-new (to finish the product)

- **WebContainer node_modules snapshot** so previews skip cold installs.

- **`app_access` tracking** so the platform "your apps" dashboard sees AppMaker usage.
- **Finish design-system migration**: remaining legacy `.scss` pages в†’ design tokens.
